DevShelf: Distributed Information Retrieval System

December 25, 2025 ¬∑ 2 min |  View Source Code

‚ö° System Overview#

DevShelf is a vertical search engine engineered to index and rank Computer Science literature without external dependencies (no Lucene/Elasticsearch).

Unlike typical student projects that scan files linearly, DevShelf implements a Split-Architecture approach:

  1. Offline Layer (IndexerMain): Pre-computes Inverted Indices and TF-IDF vectors.
  2. Online Layer (BookSearchEngine): serves sub-millisecond queries using memory-mapped data structures.

üèóÔ∏è Core Architecture (The “Torvalds” Logic)#

1. Offline Search Indexing#

The system treats data ingestion as a batch process to ensure zero latency during user queries.

  • The Indexer: IndexBuilder traverses the document corpus, applying a custom NLP Pipeline (TextProcessor) that handles Tokenization $\to$ Stop-Word Removal $\to$ Stemming.
  • Data Structure: Constructs a Positional Inverted Index serialized to JSON, allowing O(1) retrieval complexity.
  • Vectorization: Pre-calculates TF-IDF (Term Frequency-Inverse Document Frequency) scores for the entire corpus to normalize term importance.

2. Intelligent Query Processing#

The online engine (QueryProcessor) utilizes a multi-stage retrieval strategy:

  • Lexical Matching: Retrieves candidate documents using the Inverted Index.
  • Fuzzy Logic: Implements Levenshtein Distance to handle typos (e.g., “Pyton” $\to$ “Python”).
  • Predictive Search: A Trie (Prefix Tree) data structure provides O(L) autocomplete suggestions as the user types.

üß† The Hybrid Ranking Algorithm#

I engineered a custom scoring formula in the ReRanker class. It moves beyond simple keyword matching by injecting User Behavior Analytics into the equation.

The relevance score $S$ for a document $d$ and query $q$ is calculated as:

$$ S(d,q) = 0.6 \cdot \text{TF-IDF}(d,q) + 0.2 \cdot \text{Popularity}(d) + 0.2 \cdot \text{Rating}(d) $$

  • TF-IDF: The mathematical relevance of the text.
  • Popularity: Derived from LogAnalyzerMain, which processes user clickstream logs (logs.json) to quantify trending books.
  • Rating: Normalized user quality scores.

üîÑ The Data Flywheel (Feedback Loops)#

DevShelf gets smarter with use.

  1. Capture: The LoggingService records user interactions (Search, Click, Download).
  2. Analyze: LogAnalyzerMain runs offline batch jobs to update popularity.json.
  3. Adapt: The Graph module uses this data to generate context-aware recommendations (“Users who read X also read Y”).

üöÄ Impact & Scale#

  • Performance: Achieved <50ms query latency on a 78MB dataset.
  • Distribution: Engineered a “Serverless CDN” using GitHub raw content for dynamic index updates.
  • Adoption: Deployed to 200+ students via Inno Setup; captured 500+ search intents via the “Request Book” pipeline.
